# NanMuZ's Schedule - 项目开发计划

## 一、 项目背景与概述
本项目旨在开发一个可以在浏览器中访问的个人日程管理与展示应用，命名为 **"NanMuZ's Schedule"**。应用将采用纯前端静态网站架构，完美适配 GitHub Pages (`github.io`) 的免费托管。应用包含简易的密码鉴权机制以保护隐私，并能够基于无后端形式在用户的主力电脑上持久化保存日程数据，随时记录关于时间、事件、人物及 DDL 的计划。

## 二、 技术方案与选型
1. **核心框架：** React + Vite 
   *(由于应用存在大量状态交互，如拖动进度条、日历切换等，使用 React 组件化开发能够更稳定地管理数据并拥有较好的拓展性。构建后的产物为纯静态文件，可直接部署在 github.io。)*
2. **UI 与样式：** 原生 Vanilla CSS
   *(重点打造现代、轻量、高级感的用户界面，包括毛玻璃效果、深色系排版与动态微交互动画。)*
3. **数据存储：** 浏览器 `localStorage`
   *(纯前端项目，计划数据保存在你的电脑本地浏览器中，无需配置额外的数据库后端即可实现数据的随时更改。)*
4. **第三方接口：** 免费天气 API (Open-Meteo)
   *(无需 API Key 即可获取近期的天气数据，默认查询坐标设为北京市海淀区清华大学，为你带来即时气象参考。)*

## 三、 核心模块拆解
### 1. 安全访问模块 (Auth)
- 一个简易的落地页密码校验。
- 采用设备识别方案（利用浏览器缓存验证该设备是否受信任，不用每次访问都输入密码）。

### 2. 全局导航与呈现 (Header)
- **品牌 Logo：** 我们将专门设计一枚 "NanMuZ's Schedule" 标志置于显著位置。
- **全局控制：** 包含视图切换（7天视图 ↔ 模块化月视图）和基本设置（如密码配置、地点切换等预留功能）。

### 3. 日程核心视图组件 (Main Views)
- **流式周视图 (Weekly View)：**
  - 横向或交错排版一次性展示连续 5 到 7 天的时间线。
  - **“今天”视角：** 会被特殊高亮，并默认悬停于中轴或列表最顶部。
  - 每列上方标注：公历日期、星期几，以及基于 API 抓取的天气状况和温度。
- **总体月视图 (Monthly View)：**
  - 按月总览的挂历式网格排列，对长期计划一目了然。

### 4. 动态任务卡片 (Plan Card)
- **信息展示区块：** 包含 **时间 (Time), 事件 (Event), 人物 (Person), 截止日期 (DDL)** 四个明确部分。
- **三大状态交互：**
  - **未完成 (Uncompleted)：** 默认状态。
  - **进行中 (In Progress)：** 呼出进度条组件，允许鼠标自由拖动记录具体完成百分比。
  - **已完成 (Completed)：** 数据归档，并带有特殊的完成动效（如任务划线与置灰）。

## 四、 开发执行路线 (Roadmap)
- [ ] **Step 1 - 地基搭建：** 在当前目录下初始化项目环境，建立标准的现代 Web CSS 规范。
- [ ] **Step 2 - 品牌与鉴权：** 为项目设计专属 Logo；编写并植入本地登录缓存机制。
- [ ] **Step 3 - 数据与状态层：** 确立基础的日程数据结构和 LocalStorage 数据绑定。
- [ ] **Step 4 - 天气集成与周视图：** 对接清华大学位置天气，并完成 7 天流式页面的前端重构。
- [ ] **Step 5 - 卡片与交互核心：** 实装具备拖动进度条的动态任务卡及其三种状态切换功能。
- [ ] **Step 6 - 完善月视图及部署：** 进行细节动效优化，打包输出最终网站，提供 GitHub Pages 上线指导。
