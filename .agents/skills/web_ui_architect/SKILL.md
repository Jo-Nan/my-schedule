# Web UI Architect Skill

## Core Principles
1. **Visual Excellence (Aesthetics First)**
   - The user must be wowed. Avoid generic designs.
   - Use curated color palettes (e.g., sleek dark modes, vibrant HSL gradients).
   - Use dynamic animations, such as smooth hover transitions, micro-interactions, and glassmorphism effects.

2. **Structural Integrity**
   - Break UI into small, reusable, and composable elements.
   - Employ semantic HTML to improve accessibility and SEO automatically.
   - Maintain clear directory structures (e.g., separating components, pages, hooks, utilities).

3. **Performance & Responsiveness**
   - All designs must adapt effortlessly across Desktop, Tablet, and Mobile breakpoints.

## Standard Execution Workflow
When invoking this skill to create a new page or component:
1. **Analyze Requirements**: Clarify the specific functionality and brand guidelines.
2. **Build the Foundation**: Define the design tokens (colors, fonts, spacings) in a central CSS file or Tailwind config.
3. **Implement Components**: Write the necessary code, applying complex styling correctly.
4. **Integrate and Polish**: Assemble the pages, verify spacing/alignment, and add subtle animations (e.g., loading states, button ripples).

## SEO & Accessibility Best Practices
- Every main page must include standard metadata (`<title>`, `<meta name="description">`).
- Utilize a single robust `<h1>` tag per page.
- Apply ARIA tags for modals, interactive widgets, and complex navigation.
